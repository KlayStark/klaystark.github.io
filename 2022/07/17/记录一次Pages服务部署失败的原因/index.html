记录一次Pages服务部署失败的原因

发表于 2022-07-17 阅读次数： Valine：
本文字数： 631 阅读时长 ≈ 1 分钟

最近在迁移此Github+Hexo博客时踩了很多坑。

博客是使用hexo-deployer-git这个插件来将本地生成的静态博客发送到远程仓库的。

在本地在执行hexo g后，会在博客根目录下生成一个public文件夹，这个文件夹里的文件组合起来就是一个完整的静态博客。

接着如果执行hexo d，就会把这个public文件夹的东西完完整整拷贝到.deploy_git文件夹里，然后会把该文件夹里的所有文件全部推送push到远程库。之后会触发Pages服务的钩子去build项目，然后部署到网站上。

在Github Actions中报错如下：

Logging at level: debug
Configuration file: /github/workspace/./_config.yml
Theme: next
github-pages 227 | Error:  The next theme could not be found.
##[debug]Docker Action run completed with exit code 1
##[debug]Finishing: Build with Jekyll

打开.deploy_git文件夹，发现很正常，接着查看远程库里的文件，终于发现了问题。

在远程库的分支里，根本就没有hexo相关的文件，反复折腾过后，至此算是找到原因了。

解决方法

直接删除本地博客项目的.deploy_git文件夹，重新部署。

hexo clean
hexo g
hexo d
